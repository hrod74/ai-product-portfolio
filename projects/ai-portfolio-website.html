AI Portfolio Website Build

Designing and building a public portfolio site to tell my product leadership story and demonstrate practical AI-assisted development workflows.

This portfolio website is both the container for my work and a project in its own right. I built it to create a public home for my product leadership experience, AI-assisted projects, resume, and case studies while also practicing a modern AI development workflow with Codex, Claude Code, Lovable, GitHub, and static web deployment.

Problem

A standard resume could not show the full story.

My career story spans product leadership, mobile apps, real estate technology, healthcare, media, marketplaces, experimentation, and AI-assisted workflows. A standard resume could not show the full picture. I needed a site that could make the story easy to understand, show real projects, and give hiring managers a deeper view into how I think, build, and use AI.

Background Story

A prototype became a practical rebuild.

The project started as an exploration of low-cost portfolio options while I was actively searching for my next role. I considered GitHub Pages, Netlify, and other hosting paths, then used Lovable to quickly explore a visual direction. The Lovable prototype gave me a strong starting point, but export limitations meant I rebuilt the site locally as a clean static website with HTML, CSS, and JavaScript.

That constraint became part of the project. Instead of treating the AI-generated prototype as the final product, I used it as a design reference, rebuilt the site from scratch, refined the structure with Codex, and iterated section by section with product judgment, QA, and content review.

My Role

End-to-end product execution and storytelling.

I led the project end to end, including product positioning, content strategy, design direction, AI prompt design, visual QA, code review, GitHub setup, local development workflow, and deployment planning.

How I Used AI

AI supported strategy, design, content, build, and QA.

Strategy and positioning

Used ChatGPT to clarify the portfolio goal, target audience, project hierarchy, and how to position myself as an AI-enabled product leader.

Design exploration

Used Lovable to generate an initial visual direction inspired by clean product leader and editorial portfolio sites.

Rebuild from scratch

Used Codex to recreate the site locally as a lightweight static website after the Lovable version could not be exported on the free tier.

Content iteration

Used AI to turn raw career and project notes into portfolio-ready case study sections while keeping the tone human, accurate, and privacy-safe.

Code execution

Used Codex and Claude Code to build and revise the HTML, CSS, JavaScript, responsive layout, image handling, project cards, and case study pages.

QA and refinement

Reviewed desktop and mobile screenshots, identified issues like hero image cropping, awkward typography, cramped card layouts, and long-scroll case study pages, then turned those findings into targeted AI coding prompts.

GitHub workflow

Created a GitHub repo, connected the local project, committed changes in stages, and used GitHub as the source of truth for the site.

Process

From idea to static portfolio system.

  1. Defined the site goal and target audience.
  2. Reviewed example portfolio sites for design and content inspiration.
  3. Prompted Lovable to create an initial visual prototype.
  4. Rebuilt the site locally when direct export was not available.
  5. Created the GitHub repo and connected the local project.
  6. Built the homepage, hero, project cards, impact section, about section, resume section, and contact section.
  7. Added a real hero portrait and adjusted responsive image cropping.
  8. Refined typography to feel more senior, editorial, and polished.
  9. Created dedicated case study pages for the featured projects.
  10. Iterated content section by section to avoid generic AI copy.
  11. Prepared the site for GitHub Pages deployment.

Outcome

A lightweight public storytelling system.

The result is a lightweight, low-cost portfolio website that I can use in applications, recruiter conversations, and interviews. It gives each project room to explain the problem, role, process, tools, AI workflow, outcome, and lessons learned. It also demonstrates that I can use AI tools to move from strategy to shipped artifact while keeping human judgment, content accuracy, and QA in the loop.

What I Learned

AI works best inside a structured product workflow.

This project reinforced that AI is most useful when treated as a collaborator inside a structured workflow. Lovable helped with visual exploration, ChatGPT helped with product direction and prompts, and Codex helped execute code changes. But the quality came from reviewing the output, making product decisions, checking accuracy, and iterating like a product manager.
