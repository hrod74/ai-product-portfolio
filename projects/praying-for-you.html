Praying For You Mobile App Rebuild

Rebuilding a 14-year-old JavaScript prayer request prototype into a modern React Native mobile app with AI-assisted planning, Firebase, and a path to public beta.

Praying For You started as my final project in an Advanced JavaScript certification course roughly 14 years ago. The original concept was a simple community prayer request app where someone could leave a prayer request and pray for someone else’s request. Years later, I revisited the idea with modern AI tools and began evolving it from an old single-page prototype into a native mobile prayer journal and community prayer app.

Role
Product strategy, AI workflow design, technical planning, mobile UX direction, QA, and beta readiness
Tools
ChatGPT, Codex, Claude Code, Cursor, Firebase, Expo Go, React Native, GitHub
Outcome
Working React Native prototype with a clearer product direction and path toward beta

Problem

Turning a learning prototype into a real mobile product direction.

The original project was a learning prototype, not a production-ready product. It had the core idea, but it was not built for modern mobile use, app store deployment, user profiles, scalable data, or a polished prayer experience. I wanted to explore whether the concept could become a real mobile product, potentially with future monetization, while keeping the experience calm, private, and simple.

Background Story

An old course project, revisited with modern tools.

Being the curious learner that I am, I first built the original Praying For You app during an Advanced JavaScript certification course. The idea was simple: write what you are carrying, submit it as a prayer request, and pick up someone else’s request to pray for. The first version proved the concept, but it stayed as a basic web prototype for years.

With today’s AI tools, I came back to the idea and asked whether it could become something more useful. I used AI to audit the old project, rethink the product direction, and plan a path from prototype to production-ready native app for iOS and Android.

Before

The original web prototype.

The old prototype helped prove the concept, but it was not built for modern native mobile patterns, scalable data, or app store readiness.

My Role

Leading the rebuild from concept to beta readiness.

I am leading the rebuild end to end, including product strategy, AI workflow design, technical planning, mobile UX direction, Firebase setup direction, React Native implementation planning, QA, and beta readiness. I am also using the project to deepen my AI product development workflow across discovery, planning, build, review, and release preparation.

How I Used AI

AI helped move the idea from old prototype to structured mobile rebuild.

Project audit

Used Claude to audit the original app and assess whether the old concept could be evolved into a modern prayer journal and community prayer request app.

Product strategy

Reframed the app from a simple web form into a mobile-first prayer experience with a prayer feed, posting flow, verse content, settings, and profile creation.

AI development workflow

Created a full AI-assisted development workflow using agents, reviews, documentation, and planning prompts to move the app from prototype toward production readiness.

Technical planning

Planned the app architecture around React Native, Expo, Firebase, GitHub, and a future path to Google Play and App Store beta distribution.

Documentation

Built planning docs and implementation notes to define MVP scope, Firebase direction, beta requirements, account/profile behavior, prayer feed behavior, and future release phases.

Build and iteration

Used ChatGPT, Codex, Claude Code, and Cursor to support planning, implementation, issue review, and iteration while keeping product direction and QA hands-on.

Beta readiness

The current app is running as a working prototype through GitHub and Expo Go, with a target of moving toward a downloadable beta version for iOS and Android this month.

Process

From old prototype to working React Native app.

  1. Reviewed the original 14-year-old JavaScript prototype.
  2. Audited the existing project and product concept with AI.
  3. Reframed the product into a mobile prayer journal and community prayer request app.
  4. Defined the MVP experience around feed, prayer submission, verse, settings, and profile creation.
  5. Planned Firebase usage and mobile beta requirements.
  6. Created AI-assisted planning docs, review workflows, and development prompts.
  7. Built a working React Native prototype running through GitHub and Expo Go.
  8. Continued iterating toward a downloadable beta version for Google Play and the App Store.

Outcome

A working prototype with a clearer path to beta.

The project has moved from an old JavaScript learning prototype into a working React Native mobile app prototype. The current version includes key app screens such as onboarding, profile creation, prayer feed, prayer submission, verse, and settings. The project now has a clearer product direction, AI-assisted development workflow, Firebase planning, and a path toward a beta release.

What I Learned

AI is especially powerful before code is written.

This project reinforced that AI is especially powerful when used before code is written. The biggest value came from using AI to audit the old project, clarify the product strategy, plan the MVP, structure the technical path, and create repeatable review workflows. The code matters, but the product thinking around what to build, why it matters, and how to get it beta-ready matters more.
